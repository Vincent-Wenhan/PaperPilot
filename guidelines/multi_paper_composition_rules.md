# Multi-Paper Composition Rules

Classify each paper relationship as complementary, redundant, conflicting,
dependency, or alternative. Select only papers with evidence-backed product
roles.

Use one of these strategies:

- pipeline: one capability feeds the next,
- modular: capabilities remain independent behind one interface,
- alternative: select the best method for each situation,
- comparison: expose methods for side-by-side review,
- agent_workflow: capabilities collaborate through explicit stages.

Explain exclusions, integration risks, data-flow boundaries, and where mock
behavior replaces uncertain real integration.
